# Event Loop

## Самая важная концепция Node.js

Если интервьюер задаст всего один вопрос по Node.js,
то с очень большой вероятностью это будет:

```txt
Что такое Event Loop?
```

---

# Зачем вообще нужен Event Loop

JavaScript выполняется в одном потоке.

---

Например:

```js
console.log('A');
console.log('B');
console.log('C');
```

---

Результат:

```txt
A
B
C
```

---

Всё выполняется последовательно.

---

# Проблема

Представим:

```js
const data = fs.readFileSync('huge-file.txt');
```

---

Чтение занимает:

```txt
5 секунд
```

---

Что происходит?

---

Весь поток заблокирован.

---

Никакой другой код не выполняется.

---

# Решение Node.js

Асинхронность.

---

Вместо:

```js
readFileSync()
```

используем:

```js
readFile()
```

---

Тогда:

```txt
Node запускает операцию
↓
не ждёт завершения
↓
продолжает выполнять код
```

---

Но возникает вопрос:

```txt
Как понять,
когда операция завершилась?
```

---

Для этого существует Event Loop.

---

# Что такое Event Loop

Упрощенно:

```txt
while (true) {
  взять задачу
  выполнить задачу
}
```

---

На практике сложнее.

---

Event Loop постоянно проверяет:

```txt
есть ли готовые callbacks?
```

---

Если есть:

```txt
выполнить
```

---

Если нет:

```txt
ждать
```

---

# Основная идея

JavaScript выполняет код.

---

Асинхронные операции уходят:

```txt
OS
libuv
Thread Pool
```

---

Когда операция завершилась:

```txt
callback попадает в очередь
```

---

Event Loop забирает callback.

---

Выполняет.

---

# Архитектура

```txt
Call Stack
    ↓
Event Loop
    ↓
Task Queues
```

---

# Call Stack

Стек выполнения функций.

---

Например:

```js
foo();
```

---

Функция попадает в стек.

---

Когда завершается:

```txt
удаляется из стека
```

---

# Очень важное правило

Пока стек не пуст:

```txt
Event Loop ничего не выполняет
```

---

Пример:

```js
while(true){}
```

---

Event Loop никогда не получит управление.

---

Сервер зависнет.

---

# Основные фазы Event Loop

Упрощенно:

```txt
Timers
↓
Pending Callbacks
↓
Idle / Prepare
↓
Poll
↓
Check
↓
Close Callbacks
```

---

# Timers

Выполняются:

```js
setTimeout()
setInterval()
```

---

# Pending Callbacks

Некоторые системные callbacks.

---

Редко спрашивают подробно.

---

# Poll

Самая важная фаза.

---

Здесь обрабатывается:

```txt
I/O
Database
Network
Filesystem
```

---

Большая часть работы Node происходит здесь.

---

# Check

Выполняются:

```js
setImmediate()
```

---

# Close Callbacks

Например:

```js
socket.on('close')
```

---

# Упрощенная схема

```txt
Timers
↓
Poll
↓
Check
↓
Timers
↓
Poll
↓
Check
```

---

# Пример

```js
setTimeout(() => {
  console.log('timeout');
}, 0);

console.log('sync');
```

---

Результат:

```txt
sync
timeout
```

---

Почему?

---

Потому что сначала выполняется весь синхронный код.

---

Только потом Event Loop начинает выполнять очереди.

---

# Почему setTimeout(0) не равно мгновенно

Очень популярный вопрос.

---

Это означает:

```txt
не раньше чем через 0 ms
```

---

Но не:

```txt
сразу сейчас
```

---

Event Loop всё равно должен дойти до Timers Phase.

---

# Блокировка Event Loop

Очень важная тема.

---

Плохо:

```js
for(let i=0; i<10000000000; i++) {}
```

---

Что происходит?

---

Главный поток занят.

---

Event Loop не работает.

---

API перестает отвечать.

---

# Как избежать блокировки

Использовать:

```txt
Worker Threads
Queues
Streams
Chunk Processing
```

---

# Interview Answer

Event Loop — это механизм координации асинхронных операций в Node.js. Он постоянно проверяет очереди готовых задач и выполняет их, когда стек вызовов становится пустым. Благодаря Event Loop Node может эффективно обрабатывать большое количество I/O операций, не создавая отдельный поток на каждый запрос.
