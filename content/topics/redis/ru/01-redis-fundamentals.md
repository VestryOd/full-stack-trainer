# Redis Fundamentals

## Что такое Redis

Redis расшифровывается как:

```txt
Remote Dictionary Server
```

---

Redis — это:

```txt
In-Memory Data Store
```

---

Очень важно.

---

Redis:

```txt
не просто cache
```

---

Он может использоваться как:

```txt
Cache

Database

Message Broker

Queue

Pub/Sub System
```

---

# Почему Redis такой быстрый

Самый популярный вопрос.

---

Причина №1

Все данные находятся в памяти.

---

PostgreSQL:

```txt
RAM
+
Disk
```

---

Redis:

```txt
RAM
```

---

Нет обращения к диску
для большинства операций.

---

# Причина №2

Однопоточная модель.

---

Очень любят спрашивать.

---

Redis использует:

```txt
Single Threaded Event Loop
```

---

Похожая идея на Node.js.

---

# Почему один поток не тормозит

Потому что операции:

```txt
очень короткие
```

---

Например:

```txt
GET key
SET key
INCR counter
```

---

Выполняются за микросекунды.

---

Нет блокировок.

---

Нет race conditions внутри Redis.

---

# Причина №3

Оптимизированные структуры данных.

---

Например:

```txt
Hash Table

Skip List

Linked List
```

---

Под конкретные операции.

---

# Redis как Cache

Самый популярный сценарий.

---

Flow:

```txt
Request
 ↓
Redis
 ↓
Miss
 ↓
PostgreSQL
 ↓
Redis
 ↓
Response
```

---

# Cache Hit

Данные найдены в Redis.

---

К БД не идем.

---

# Cache Miss

Данных нет.

---

Читаем из БД.

---

# TTL

Time To Live.

---

Очень важная тема.

---

Пример:

```bash
SET user:1 value EX 60
```

---

Через:

```txt
60 секунд
```

---

Ключ удалится.

---

# Eviction

Очень популярный вопрос.

---

Что делать если память закончилась?

---

Redis поддерживает политики:

```txt
LRU

LFU

TTL
```

---

# LRU

Least Recently Used.

---

Удаляем:

```txt
наименее используемые ключи
```

---

# Частый вопрос

Почему Redis такой быстрый?

Ответ:

Потому что хранит данные в памяти, использует однопоточный Event Loop и оптимизированные структуры данных.

---

# Частый вопрос

Можно ли использовать Redis вместо PostgreSQL?

Ответ:

Для большинства бизнес-систем нет. Redis обычно используется как дополнительный слой кеширования поверх основной базы данных.

---

# Interview Answer

Redis — это высокопроизводительное in-memory хранилище данных, которое чаще всего используется для кеширования, хранения сессий, счетчиков и реализации очередей. Высокая скорость достигается за счет хранения данных в памяти и использования однопоточной модели выполнения.
