# XSS, CSRF и CORS

## Самая популярная тема по Web Security

Очень часто спрашивают:

```txt
Что такое XSS?

Что такое CSRF?

Что такое CORS?

Чем они отличаются?
```

---

# XSS

Cross Site Scripting.

---

Самая распространенная веб-атака.

---

# Идея атаки

Злоумышленник заставляет браузер выполнить:

```html
<script>
malicious code
</script>
```

---

На вашем сайте.

---

# Пример

Пользователь пишет комментарий:

```html
<script>
alert('Hacked')
</script>
```

---

Сайт сохраняет его в БД.

---

Другой пользователь открывает страницу.

---

Браузер выполняет:

```html
<script>
...
</script>
```

---

# Получаем

```txt
XSS
```

---

# Почему это опасно

Можно украсть:

```txt
JWT

Cookies

LocalStorage

User Actions
```

---

# Stored XSS

Самый опасный вариант.

---

Код сохраняется в БД.

---

Пример:

```txt
Комментарии

Форум

Чат
```

---

# Flow

```txt
Attacker
 ↓
Database
 ↓
Victim
```

---

# Reflected XSS

Не сохраняется.

---

Передается через запрос.

---

Например:

```txt
/search?q=<script>...</script>
```

---

Сервер возвращает его обратно.

---

Возникает на клиенте.

---

Пример:

```js
element.innerHTML =
 userInput;
```

---

Очень популярный вопрос.

---

# Как защититься

Самый важный вопрос.

---


---

---

```js
textContent
```



# CSRF

Cross Site Request Forgery.

---

Следующая популярная атака.

---

# Идея

Пользователь уже авторизован.

---

Злоумышленник заставляет браузер:

```txt
отправить запрос
от имени пользователя
```

---

# Пример

Пользователь открыт:

```txt
bank.com
```

---

И одновременно:

```txt
evil.com
```

---

На evil.com есть:

```html
<form
 action="bank.com/transfer"
 method="POST">
</form>
```

---

Браузер автоматически приложит:

```txt
Cookies
```

---

Банк думает:

```txt
это настоящий пользователь
```

---

# Почему JWT в Header защищает

Очень популярный вопрос.

---

Браузер автоматически отправляет:

```txt
Cookies
```

---

Но не отправляет:

```txt
Authorization Header
```

---

Поэтому:

```txt
JWT Header
```

---

Сильно снижает риск CSRF.

---

# CSRF Token

Классическая защита.

---

Сервер выдает:

```txt
случайный токен
```

---

Фронтенд отправляет его обратно.

---

Сервер проверяет.

---

---

На самом деле:

```txt
CORS
НЕ защищает сервер
```

---

# Что такое CORS

Cross-Origin Resource Sharing.

---

Политика браузера.

---

# Пример

Frontend:

```txt
localhost:3000
```

---

Backend:

```txt
api.com
```

---

```txt
Different Origin
```

---

---

---

# Частый вопрос

Защищает ли CORS API?

Ответ:

Нет.

---

Он защищает браузер.

---

Сервер по-прежнему можно вызвать:

```txt
curl

Postman

другой backend
```

---

# Частый вопрос

Чем XSS отличается от CSRF?

Ответ:

XSS заставляет браузер выполнять чужой JavaScript.

CSRF заставляет браузер отправлять легитимный запрос от имени пользователя.

---

# Частый вопрос

Почему HttpOnly помогает против XSS?

Ответ:

JavaScript не может прочитать содержимое cookie.

---

# Interview Answer

XSS позволяет внедрить и выполнить вредоносный JavaScript на сайте. CSRF заставляет браузер отправлять запросы от имени авторизованного пользователя. CORS является механизмом браузера, регулирующим междоменные запросы, но не является полноценным механизмом защиты API.
