# URL Shortener Design

## Постановка задачи

Нужно реализовать:

```txt
bit.ly

tinyurl

goo.gl
```

---

Пример.

---

Вход:

```txt
https://google.com/very/long/url
```

---

Выход:

```txt
https://app.com/abc123
```

---

---

создавать короткие ссылки

считать статистику
```

---

 ↓
Database
```

---

# Частый вопрос

Почему используют Redis?

Ответ:

Большинство запросов — редиректы уже существующих ссылок.

---

# Частый вопрос

Почему Base62?

Ответ:

Позволяет компактно кодировать числовой ID.

---

# Частый вопрос

Почему аналитика через очередь?

Ответ:

Чтобы не увеличивать latency редиректа.

---

# Interview Answer

URL Shortener обычно строится на PostgreSQL для хранения ссылок, Redis для кеширования популярных URL и очередях для асинхронной обработки аналитики. Для генерации коротких кодов часто используется Base62-кодирование автоинкрементного идентификатора.
