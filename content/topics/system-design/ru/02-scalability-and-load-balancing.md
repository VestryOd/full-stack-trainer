# Scalability и Load Balancing

## Масштабирование

Самая частая тема System Design.

---

# Вертикальное масштабирование

Scale Up.

---

Увеличиваем ресурсы сервера.

---

```txt
8 CPU
 ↓
32 CPU
```

---

# Плюсы

```txt
простота
```

---

# Минусы

```txt
есть предел
```

---

# Горизонтальное масштабирование

Scale Out.

---

Добавляем новые серверы.

---

```txt
Server A

Server B

Server C
```

---

# Частый вопрос

Когда вертикальное масштабирование перестает работать?

Ответ:

Когда сервер достигает физических ограничений CPU, RAM или диска.

---

# Частый вопрос

Зачем нужен Load Balancer?

Ответ:

Для распределения нагрузки и повышения отказоустойчивости.

---

# Частый вопрос

Почему Stateless сервисы проще масштабировать?

Ответ:

Потому что любой запрос можно обработать на любом сервере.

---

# Interview Answer

Масштабирование может быть вертикальным или горизонтальным. В современных системах чаще используется горизонтальное масштабирование через Load Balancer и несколько stateless инстансов приложения.
