# CSS Grid: глубокое погружение

## Фундаментальное отличие от Flexbox

Flexbox — это **от контента наружу**: контейнер адаптируется под элементы. Вы определяете размеры и правила роста элементов, контейнер их вмещает.

Grid — это **от раскладки внутрь**: контейнер определяет жёсткую двухмерную структуру треков (строк и колонок), а элементы размещаются в ячейках этой структуры. Сетка существует независимо от своих элементов.

Именно это различие определяет, когда выбирать один или другой — но подробнее об этом в конце.

## Явная и неявная сетка

**Явная сетка** — то, что вы определяете через `grid-template-columns`, `grid-template-rows` и `grid-template-areas`. **Неявная сетка** — то, что браузер создаёт автоматически, когда элементы выходят за границы явной сетки.

```css
.container {
  display: grid;
  grid-template-columns: 200px 1fr 1fr; /* 3 явные колонки */
  grid-template-rows: 80px auto;        /* 2 явные строки */
}
```

Если элементы требуют третьей строки, браузер создаёт её автоматически — это неявная сетка. Размер неявных треков задаётся через `grid-auto-rows` и `grid-auto-columns`:

```css
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: 200px;    /* только одна явная строка */
  grid-auto-rows: 150px;        /* все последующие неявные строки: 150px */
}
```

Без `grid-auto-rows` неявные строки получают размер `auto` — сжимаются до высоты содержимого. Это источник ошибок "почему вторая строка такая низкая?".

`grid-auto-flow` управляет тем, как элементы размещаются в неявной сетке:

```css
grid-auto-flow: row;    /* по умолчанию: сначала заполняются строки, затем добавляются новые */
grid-auto-flow: column; /* сначала заполняются колонки, затем добавляются новые */
grid-auto-flow: dense;  /* заполняет пустоты, оставшиеся от крупных элементов — меняет визуальный порядок */
```

`grid-auto-flow: dense` полезен для masonry-подобных раскладок, но меняет визуальный порядок элементов, что нарушает клавиатурную навигацию и работу скринридеров. Используйте с осторожностью.

## Единица `fr` — что это на самом деле

`fr` — **доля доступного свободного пространства**, а не доля общей ширины контейнера. Это различие важно, когда сетка содержит треки с фиксированным размером рядом с `fr`-треками.

```css
.container {
  display: grid;
  width: 900px;
  grid-template-columns: 200px 1fr 2fr;
}
```

Разрешение:
1. Вычесть фиксированные треки: `900 - 200 = 700px` доступно
2. Сумма долей: `1 + 2 = 3`
3. Одна `fr` = `700 / 3 ≈ 233px`
4. Колонка 2: `1fr = 233px`, Колонка 3: `2fr = 466px`

```css
/* fr с gap */
.container {
  display: grid;
  width: 900px;
  gap: 20px;
  grid-template-columns: 1fr 1fr 1fr;
}
/* Сначала вычитаются gap: 900 - (2 × 20) = 860px доступно
   Каждая колонка: 860 / 3 ≈ 286px */
```

`fr` вычисляется после разрешения всех фиксированных треков (px, em, %, min-content, max-content) и gap. Оставшееся пространство делится на доли.

**`fr` vs `%`:**

```css
/* % включает пространство, занятое gap — колонки переполняются */
grid-template-columns: 33.33% 33.33% 33.33%;
/* С gap: 3 × 33.33% + 2 × 20px > 100% — переполнение */

/* fr исключает gap — всегда вписывается */
grid-template-columns: 1fr 1fr 1fr;
/* С gap: всегда корректно делит оставшееся пространство */
```

Именно поэтому `fr` предпочтительнее `%` для пропорциональных колонок.

**`minmax()` с `fr`:**

```css
/* Колонка не сужается ниже 200px, но может расти, заполняя свободное место */
grid-template-columns: minmax(200px, 1fr) minmax(200px, 2fr);
```

## `grid-template-areas` — читаемые раскладки

`grid-template-areas` присваивает имена прямоугольным областям сетки и позволяет помещать элементы в них по имени. Результат — самодокументирующаяся раскладка:

```css
.layout {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: 64px 1fr 48px;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
  min-height: 100vh;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }
```

Правила для `grid-template-areas`:
- Каждая строка — это строка в кавычках; каждое слово — имя ячейки
- Именованная область должна быть прямоугольной — L-образные формы недопустимы
- Используйте `.` для безымянной ячейки (дыры)
- Каждая строка должна содержать одинаковое количество ячеек

```css
/* Корректно: header охватывает обе колонки, средняя строка — две области, footer охватывает обе */
grid-template-areas:
  "header header"
  "sidebar main"
  "footer  footer";

/* Некорректно: непрямоугольная именованная область */
grid-template-areas:
  "header main"
  "header footer"; /* "header" появляется в строках 1 и 2, но только в колонке 1 — L-форма */

/* Корректно: дыра в сетке */
grid-template-areas:
  "logo    nav    nav"
  ".       main   sidebar"
  "footer  footer footer";
```

Установка `grid-area` на элементе неявно размещает его, под капотом устанавливая `grid-column` и `grid-row` в соответствии с границами области.

## Размещение элементов — номера линий, span, именованные линии

Линии сетки нумеруются с 1 (или с -1 с конца). Элементы размещаются указанием начальной и конечной линий:

```css
.item {
  grid-column: 1 / 3;    /* от линии 1 до линии 3 (2 колонки) */
  grid-row: 2 / 4;       /* от линии 2 до линии 4 (2 строки) */
}

/* Эквивалент с использованием span */
.item {
  grid-column: 1 / span 2; /* начать с линии 1, охватить 2 колонки */
  grid-row: 2 / span 2;
}

/* С конца */
.item {
  grid-column: 1 / -1; /* от первой до последней линии (полная ширина) */
}
```

Именованные линии позволяют семантическое размещение:

```css
.container {
  grid-template-columns:
    [sidebar-start] 240px [sidebar-end main-start] 1fr [main-end];
}

.main-content {
  grid-column: main-start / main-end;
}
```

`grid-template-areas` автоматически создаёт именованные линии: если область называется `header`, неявно создаются линии `header-start` и `header-end`.

## `auto-fill` vs `auto-fit` — тонкое различие

Оба используются с `repeat()` для создания максимально возможного количества треков. Разница проявляется только тогда, когда элементов недостаточно для заполнения всех треков.

```css
/* auto-fill: создаёт максимально возможное количество треков, даже если они пустые */
grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

/* auto-fit: создаёт максимально возможное количество треков, затем схлопывает пустые в 0 */
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
```

**Визуальное различие при малом количестве элементов:**

```
Контейнер: 900px, minmax(200px, 1fr), 3 элемента

auto-fill:
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│  A   │ │  B   │ │  C   │ │пусто │
└──────┘ └──────┘ └──────┘ └──────┘
A, B, C получают по 225px (900/4 трека)
4-й трек пустой, но всё равно занимает место

auto-fit:
┌────────────┐ ┌────────────┐ ┌────────────┐
│     A      │ │     B      │ │     C      │
└────────────┘ └────────────┘ └────────────┘
Пустые треки схлопываются в 0
A, B, C получают по 300px (900/3 трека)
```

**Когда что использовать:**

- `auto-fill`: когда нужно, чтобы элементы сохраняли минимальный размер, даже если остаются пустые колонки — полезно при динамическом добавлении элементов без рефлоу раскладки
- `auto-fit`: когда нужно, чтобы элементы росли и заполняли всю строку — типичный сценарий "адаптивная сетка без медиазапросов"

```css
/* Адаптивная сетка без медиазапросов — наиболее распространённый сценарий */
.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 16px;
}
/* Результат: максимальное количество колонок при минимуме 280px, всегда заполняющих полную ширину */
```

## `minmax()`, `min-content` и `max-content`

`minmax(min, max)` определяет диапазон размера трека. Браузер назначает любой размер между `min` и `max`:

```css
/* Колонка шириной не менее 200px, но может расти до 1fr */
grid-template-columns: minmax(200px, 1fr);

/* Строка высотой не менее высоты контента, но не более 300px */
grid-template-rows: minmax(min-content, 300px);
```

`min-content`: наименьший размер, при котором контент не переполняется (ширина самого длинного слова для текста, интринсическая ширина для изображений).
`max-content`: размер, при котором контент вообще не переносится (полное предложение в одну строку).

```css
/* Колонка по размеру самого длинного слова, без переноса */
grid-template-columns: min-content 1fr;

/* Колонка по размеру самой длинной строки контента */
grid-template-columns: max-content 1fr;
```

`fit-content(value)` ограничивает трек значением `value`, но позволяет ему сужаться до `min-content`:

```css
/* Трек растёт до 300px, сужается до минимума контента */
grid-template-columns: fit-content(300px) 1fr;
```

## Subgrid — выравнивание через границы компонентов

Стандартные grid-элементы создают собственный контекст форматирования — колонки дочерней сетки не зависят от колонок родительской. Subgrid позволяет дочернему grid-элементу наследовать структуру треков родителя:

```css
.parent {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-template-rows: auto auto auto;
  gap: 16px;
}

.card {
  grid-column: span 2;
  display: grid;
  /* Вместо определения новых колонок — наследуем треки родителя */
  grid-template-columns: subgrid;
  /* Наследуем строковые треки родителя */
  grid-template-rows: subgrid;
}

.card-header { grid-row: 1; }
.card-body   { grid-row: 2; }
.card-footer { grid-row: 3; }
```

С subgrid заголовки всех карточек выравниваются по одной строке, контентные части — по другой, футеры — по третьей, даже если высота контента у карточек разная. Без subgrid каждая карточка — независимая сетка, и выровнять строки между карточками без JavaScript невозможно.

Поддержка браузерами: subgrid поддерживается всеми основными браузерами (Chrome 117+, Firefox 71+, Safari 16+). Долгое время это была наиболее востребованная отсутствующая возможность CSS.

## Именованные линии и области для адаптивных раскладок

Мощный паттерн: определить несколько вариантов раскладки через единственное переопределение `grid-template-areas` в медиазапросе:

```css
.layout {
  display: grid;
  gap: 16px;
  grid-template-columns: 1fr;
  grid-template-areas:
    "header"
    "main"
    "sidebar"
    "footer";
}

@media (min-width: 768px) {
  .layout {
    grid-template-columns: 240px 1fr;
    grid-template-areas:
      "header  header"
      "sidebar main"
      "footer  footer";
  }
}

/* Размещение элементов не меняется — меняются только определения областей */
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }
```

Все элементы остаются привязанными к своим именованным областям. Медиазапрос меняет только позиции этих областей. Медиазапросы на уровне элементов не нужны.

## Реальные примеры раскладок

### Дашборд с фиксированным сайдбаром и динамической основной областью

```css
.dashboard {
  display: grid;
  grid-template-columns: 260px 1fr;
  grid-template-rows: 60px 1fr;
  grid-template-areas:
    "sidebar topbar"
    "sidebar content";
  height: 100vh;
}

.sidebar  { grid-area: sidebar; overflow-y: auto; }
.topbar   { grid-area: topbar; }
.content  { grid-area: content; overflow-y: auto; }
```

### Приближение masonry через плотное заполнение

```css
.masonry {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  grid-auto-rows: 10px; /* мелкая единица для точного управления строками */
  gap: 16px 16px;
  grid-auto-flow: dense;
}

/* Элементы сами контролируют высоту через охват строк */
.item-small  { grid-row: span 20; } /* 20 × 10px = 200px */
.item-medium { grid-row: span 30; }
.item-large  { grid-row: span 40; }
```

### Сетка календаря

```css
.calendar {
  display: grid;
  grid-template-columns: repeat(7, 1fr);
  gap: 1px;
  background: var(--border-color);
}

.day-header {
  background: white;
  text-align: center;
  padding: 8px;
}

.day {
  background: white;
  min-height: 100px;
  padding: 8px;
}

.day[data-weekday="2"] { grid-column: 2; } /* Первый день месяца во вторник */
```

### Журнальная раскладка с именованными областями

```css
.magazine {
  display: grid;
  grid-template-columns: repeat(6, 1fr);
  grid-template-rows: auto;
  gap: 16px;
}

.lead-story {
  grid-column: 1 / 4;
  grid-row: 1 / 3;
}

.secondary-story {
  grid-column: 4 / 7;
  grid-row: 1;
}

.tertiary-story {
  grid-column: 4 / 7;
  grid-row: 2;
}
```

## Когда Grid лучше Flexbox — и когда нет

**Используйте Grid когда:**

1. **Двухмерная раскладка** — нужно одновременно управлять строками и колонками. Макеты страниц, дашборды, сетки карточек, где высоты строк должны совпадать между колонками.

2. **Мышление от раскладки** — структура определена независимо от контента. Сетка существует, элементы заполняют её.

3. **Выравнивание между независимыми компонентами** — subgrid позволяет заголовкам, телам и футерам карточек-сиблингов выравниваться по одним трековым линиям.

4. **Именованные области** — когда `grid-template-areas` лучше передаёт смысл раскладки, чем вложенные div-ы.

5. **Точное размещение** — элементы, которым нужно перекрываться или располагаться по конкретным координатам сетки.

**Используйте Flexbox когда:**

1. **Одномерная раскладка** — элементы в строке ИЛИ в колонке, где выравнивание по поперечной оси вторично.

2. **Размер по контенту** — размеры элементов определяются содержимым, и контейнер должен их вмещать.

3. **Динамическое количество элементов** — навигационные панели, списки тегов, тулбары, где количество элементов меняется и нужен естественный перенос.

4. **Распределение пространства вдоль одной оси** — `justify-content: space-between` для шапки с логотипом и действиями.

5. **Внутренности компонента** — внутренняя раскладка карточки (иконка + текст + кнопка) почти всегда Flexbox.

**Практическое правило:**

Раскладка страницы, сетки, всё где строки и колонки взаимодействуют → Grid.
Внутренности компонентов, линейные списки, распределения по одной оси → Flexbox.

Они естественно комбинируются:

```css
/* Grid для раскладки страницы */
.page { display: grid; grid-template-areas: "sidebar main"; }

/* Flexbox внутри каждой grid-области для внутренностей компонента */
.main-header { display: flex; align-items: center; justify-content: space-between; }
```

## Типичные ошибки на интервью

**"В чём разница между `auto-fill` и `auto-fit`?"**

Оба создают максимально возможное количество треков. Разница проявляется, когда элементов меньше, чем треков. `auto-fill` сохраняет пустые треки — элементы не вырастают за пределы своего `max` в `minmax()`. `auto-fit` схлопывает пустые треки в ноль — элементы растут и заполняют всю ширину контейнера. При достаточном количестве элементов для заполнения всех колонок оба ведут себя одинаково.

---

**"Что именно означает `1fr`?"**

Одна доля доступного свободного пространства — не одна доля ширины контейнера. Свободное пространство — это то, что остаётся после разрешения всех фиксированных треков (px, em, %) и gap. Если контейнер 900px имеет фиксированную колонку 200px и две `1fr` колонки с gap 20px, свободное пространство = `900 - 200 - 2×20 = 660px`, и каждая `fr` = `330px`.

---

**"Почему `fr` лучше `%` для колонок сетки?"**

Проценты вычисляются от полной ширины контейнера, до вычитания gap. Три колонки `33.33%` с gap `20px` в сумме превышают 100% и переполняются. `fr` распределяет только оставшееся пространство после gap — колонки никогда не переполняются.

---

**"Как сделать элемент на всю ширину в сетке?"**

`grid-column: 1 / -1`. Линия `-1` всегда обозначает последнюю линию явной сетки. При `grid-template-columns: repeat(4, 1fr)` есть 5 колоночных линий (с 1 по 5). `1 / -1` охватывает от линии 1 до линии 5 — полная ширина. Важно: `-1` работает только для явной сетки. Элементы в неявной сетке (дополнительные строки) не могут использовать `-1` для полной ширины, если эти строки не определены явно в `grid-template-rows`.

---

**"Почему `grid-auto-flow: dense` не сохраняет визуальный порядок элементов?"**

Плотное заполнение закрывает дыры в сетке, оставшиеся от крупных элементов. Для этого браузер может поместить более мелкий поздний элемент перед крупным ранним. Порядок DOM (и порядок Tab) остаётся неизменным — меняется только визуальная позиция. Это создаёт несоответствие между визуальным и клавиатурным порядком, что является проблемой доступности. `dense` следует использовать только для чисто декоративных сеток (галереи изображений), где порядок клавиатурной навигации не обязан совпадать с визуальным.

---

**"Что такое subgrid и когда его использовать?"**

Subgrid позволяет grid-элементу, который сам является grid-контейнером, наследовать (а не копировать) строковые или колоночные треки родителя. Без subgrid: сетка карточек, где у каждой карточки есть заголовок, тело и футер — если у карточек разная высота контента, футеры не выравниваются между карточками. С `grid-template-rows: subgrid` все карточки делят строковые треки родителя, и футеры автоматически выравниваются. Поддерживается всеми основными браузерами начиная с Chrome 117 (август 2023).
