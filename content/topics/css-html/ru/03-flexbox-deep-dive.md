# Flexbox: глубокое погружение

## Ментальная модель: два оси, один контейнер

Flexbox работает с одной осью одновременно. **Главная ось** — направление, в котором выстраиваются элементы; **поперечная ось** — перпендикулярна ей. Каждое flexbox-свойство относится к одной из этих двух осей.

```
flex-direction: row (по умолчанию)

главная ось →→→→→→→→→→→→→→→→→→→→→→
┌─────────────────────────────────┐
│  ┌───┐  ┌───┐  ┌───┐  ┌───┐   │ ↑ поперечная ось
│  │ A │  │ B │  │ C │  │ D │   │ │
│  └───┘  └───┘  └───┘  └───┘   │ ↓
└─────────────────────────────────┘

flex-direction: column

┌─────────────────────────────────┐
│  ┌───────────────────────────┐  │ ↑ главная ось
│  │ A                         │  │ │
│  └───────────────────────────┘  │ │
│  ┌───────────────────────────┐  │ │
│  │ B                         │  │ ↓
│  └───────────────────────────┘
   → поперечная ось
```

`justify-content` всегда управляет выравниванием вдоль **главной оси**.
`align-items` / `align-self` всегда управляют выравниванием вдоль **поперечной оси**.

При изменении `flex-direction` оси поворачиваются — `justify-content: center` центрирует вертикально в колоночной раскладке. Это сбивает разработчиков, запомнивших "justify = горизонталь" — это верно только при `flex-direction: row`.

## Алгоритм flex — как браузер распределяет пространство

Прежде чем разбираться с `flex-grow`, `flex-shrink` и `flex-basis`, нужно понять порядок, в котором браузер разрешает размеры flex-элементов. Алгоритм состоит из четырёх этапов.

### Этап 1: Определение гипотетического размера по главной оси

**Гипотетический размер** каждого элемента разрешается из `flex-basis`. Порядок разрешения:

1. Если `flex-basis: auto` → используется `width` элемента (или `height` в колоночной раскладке). Если и `width: auto` → используется max-content-размер (по содержимому).
2. Если `flex-basis` — длина (`px`, `%` и т.д.) → используется именно это значение (`width`/`height` игнорируются).
3. Если `flex-basis: content` → используется max-content-размер.

```css
.item {
  flex-basis: auto;
  width: 200px;
  /* Гипотетический размер: 200px */
}

.item {
  flex-basis: 150px;
  width: 200px;
  /* Гипотетический размер: 150px — flex-basis побеждает */
}

.item {
  flex-basis: auto;
  /* width не задан → гипотетический размер = ширина по содержимому */
}
```

### Этап 2: Определение свободного пространства или переполнения

Сложить все гипотетические размеры вместе с gap. Сравнить с размером контейнера по главной оси.

- **Свободное пространство > 0**: в контейнере есть остаток → `flex-grow` его распределяет.
- **Свободное пространство < 0**: элементы переполняют контейнер → `flex-shrink` поглощает переполнение.
- **Свободное пространство = 0**: элементы вписываются точно, рост и сжатие не нужны.

### Этап 3a: Распределение свободного пространства через `flex-grow`

`flex-grow` задаёт пропорцию, а не пиксельное значение или долю итогового размера. Свободное пространство распределяется пропорционально между элементами с ненулевым `flex-grow`.

```
Контейнер: 600px
Элементы: A (flex-grow: 1, flex-basis: 0), B (flex-grow: 2, flex-basis: 0), C (flex-grow: 1, flex-basis: 0)

Сумма flex-grow: 1 + 2 + 1 = 4
Свободное пространство: 600px (flex-basis у всех равен 0)

A получает: 600 × (1/4) = 150px
B получает: 600 × (2/4) = 300px
C получает: 600 × (1/4) = 150px
```

При ненулевых значениях `flex-basis`:

```
Контейнер: 600px
A: flex-grow: 1, flex-basis: 100px
B: flex-grow: 2, flex-basis: 100px
C: flex-grow: 1, flex-basis: 100px

Сумма flex-basis: 300px
Свободное пространство: 600 - 300 = 300px (ИМЕННО ОНО распределяется)

A получает: 100 + 300 × (1/4) = 100 + 75 = 175px
B получает: 100 + 300 × (2/4) = 100 + 150 = 250px
C получает: 100 + 300 × (1/4) = 100 + 75 = 175px
```

Ключевой инсайт: `flex-grow` распределяет **свободное пространство**, а не общее. Именно поэтому `flex-grow: 1` на всех элементах НЕ означает равную ширину — это верно только если `flex-basis` одинаков для всех (включая `flex-basis: 0`).

### Этап 3b: Поглощение переполнения через `flex-shrink`

`flex-shrink` работает аналогично, но поглощает отрицательное свободное пространство. При расчёте используются **взвешенные коэффициенты сжатия** — значение `flex-shrink` умножается на `flex-basis` каждого элемента для взвешенного распределения:

```
Контейнер: 400px
A: flex-shrink: 1, flex-basis: 200px  → взвешенный коэффициент: 1 × 200 = 200
B: flex-shrink: 2, flex-basis: 200px  → взвешенный коэффициент: 2 × 200 = 400
C: flex-shrink: 1, flex-basis: 200px  → взвешенный коэффициент: 1 × 200 = 200

Сумма взвешенных коэффициентов: 800
Переполнение: (200 + 200 + 200) - 400 = 200px (нужно поглотить)

A сжимается на: 200 × (200/800) = 50px → итог: 150px
B сжимается на: 200 × (400/800) = 100px → итог: 100px
C сжимается на: 200 × (200/800) = 50px → итог: 150px
```

Это взвешивание означает, что более крупные элементы сжимаются больше (в абсолютных пикселях) при равных значениях `flex-shrink` — намеренный дизайн, чтобы элементы не сжимались непропорционально относительно своего размера.

### Этап 4: Применение ограничений min/max

После роста/сжатия браузер ограничивает каждый элемент до `min-width`/`max-width` (или `min-height`/`max-height` в колоночной раскладке). Если ограничение срабатывает, избыточное свободное пространство или переполнение перераспределяется между неограниченными элементами. Итерация продолжается до тех пор, пока ограничения больше не срабатывают.

## Ловушка `min-width: auto`

Это самый распространённый баг во Flexbox и один из наиболее часто задаваемых вопросов на интервью.

По умолчанию flex-элементы имеют `min-width: auto`. Для большинства элементов `auto` разрешается в `0`. Но для элементов с собственным контентом (текстовые узлы, изображения, элементы с явной `width` контента) `min-width: auto` разрешается в **минимальный размер контента** — наименьший размер, при котором содержимое не переполняется.

```css
.container {
  display: flex;
  width: 300px;
}

.item {
  flex-shrink: 1;
  flex-basis: 200px;
  /* min-width: auto → разрешается в минимальную ширину контента */
}
```

Если элемент содержит длинное слово или изображение фиксированной ширины, `min-width: auto` может оказаться `150px`. Даже при `flex-shrink: 1` элемент не сможет сжаться ниже `150px`. Если контейнер слишком мал — элемент переполняет его, несмотря на установленный `flex-shrink`.

**Исправление:**

```css
.item {
  min-width: 0; /* Переопределяем min-width: auto */
  overflow: hidden; /* или overflow: auto, чтобы содержать контент */
}
```

После установки `min-width: 0` свойство `flex-shrink` может сжать элемент ниже размера его контента, а `overflow: hidden` обрабатывает обрезанный контент.

Это также влияет на усечение текста:

```css
/* Не работает — flex-элемент не сожмётся, текст не усечётся */
.text-item {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

/* Работает */
.text-item {
  min-width: 0; /* ← ключевое исправление */
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

Та же проблема возникает в колоночных раскладках с `min-height: auto` на flex-элементах, содержащих изображения или имеющих явную высоту.

## Сокращение `flex` — что на самом деле устанавливает браузер

```css
/* flex: <grow> <shrink> <basis> */
flex: 1;         /* flex: 1 1 0% — НЕ flex: 1 1 auto */
flex: auto;      /* flex: 1 1 auto */
flex: none;      /* flex: 0 0 auto — полностью негибкий */
flex: 0;         /* flex: 0 1 0% */
flex: 1 200px;   /* flex: 1 1 200px */
```

Критическое различие между `flex: 1` и `flex: 1 1 auto`:

```css
/* flex: 1 → flex-basis: 0% → элементы делят ВСЁ пространство контейнера пропорционально */
.item { flex: 1; }

/* flex: auto → flex-basis: auto → элементы начинают с размера своего контента,
   затем делят только СВОБОДНОЕ пространство пропорционально */
.item { flex: auto; }
```

`flex: 1` у всех элементов → элементы равной ширины (все начинают с 0 и растут одинаково).
`flex: auto` у всех элементов → элементы с бо́льшим контентом получают бо́льше пространства.

Используйте `flex: 1`, когда нужно настоящее равное распределение.
Используйте `flex: auto`, когда ширина контента должна влиять на итоговые размеры.

## Свойства выравнивания — полная карта

```
Главная ось (justify-*):
  justify-content   → распределение пространства между/вокруг элементов в контейнере
  justify-items     → значение justify-self по умолчанию (редко используется во flex)
  justify-self      → выравнивание отдельного элемента (НЕ работает во flex — только в grid)

Поперечная ось (align-*):
  align-content     → распределение пространства между/вокруг строк (только многострочно)
  align-items       → выравнивание всех элементов по поперечной оси по умолчанию
  align-self        → переопределение для отдельного элемента
```

`justify-self` не работает во Flexbox — распространённая ошибка. Чтобы выровнять один flex-элемент по концу главной оси, используйте `margin: auto`:

```css
/* Навигация: логотип слева, ссылки справа */
.nav {
  display: flex;
  align-items: center;
}

.nav-logo { /* остаётся в начале */ }

.nav-links {
  margin-left: auto; /* выталкивает в конец главной оси */
}
```

`margin: auto` во Flexbox поглощает всё доступное свободное пространство в указанном направлении. Это flexbox-эквивалент `float: right` — без побочных эффектов float.

```css
/* Центрирование одного элемента в контейнере — по обеим осям */
.container {
  display: flex;
}
.item {
  margin: auto; /* поглощает свободное пространство во всех направлениях */
}
```

## `align-content` vs `align-items` — важное различие

`align-items` управляет выравниванием элементов внутри одной flex-строки.
`align-content` управляет тем, как несколько flex-строк распределяют пространство внутри контейнера — работает только при `flex-wrap: wrap` и наличии как минимум двух строк.

```css
.container {
  display: flex;
  flex-wrap: wrap;
  height: 400px;

  align-items: center;    /* элементы выровнены по центру внутри каждой строки */
  align-content: center;  /* сами строки выровнены по центру в 400px высоты */
}
```

В однострочном flex-контейнере (без wrap или с одной строкой) `align-content` не имеет эффекта — используйте `align-items`.

## Свойство `gap`

`gap` (ранее `grid-gap`) задаёт расстояние между flex-элементами — row-gap и column-gap:

```css
.container {
  display: flex;
  gap: 16px;        /* row-gap: 16px; column-gap: 16px */
  gap: 8px 16px;   /* row-gap: 8px; column-gap: 16px */
  row-gap: 8px;
  column-gap: 16px;
}
```

`gap` теперь поддерживается повсеместно. Ключевое преимущество перед `margin`: gap применяется только между элементами, никогда на внешних краях — не нужны хаки с отрицательными margin.

```css
/* Старый подход — некрасиво */
.container { margin: 0 -8px; }
.item { margin: 0 8px; }

/* Современный подход */
.container { display: flex; gap: 16px; }
/* Внешние margin не нужны */
```

## Реальные примеры раскладок

### Классическая holy grail раскладка

```css
.page {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

.header,
.footer {
  flex: none; /* не растут и не сжимаются — фиксированная высота */
}

.body {
  display: flex;
  flex: 1; /* растёт, заполняя оставшееся вертикальное пространство */
}

.sidebar {
  flex: 0 0 240px; /* фиксированная ширина, без роста/сжатия */
}

.main {
  flex: 1; /* занимает всё оставшееся горизонтальное пространство */
  min-width: 0; /* предотвращает поломку раскладки контентом */
}
```

### Адаптивная сетка карточек без медиазапросов

```css
.card-grid {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}

.card {
  flex: 1 1 280px;
  /* flex-basis: 280px — элементы хотят быть шириной 280px.
     flex-grow: 1 — растут, заполняя оставшееся место.
     flex-shrink: 1 — могут сжиматься ниже 280px.
     Результат: максимальное количество колонок при минимуме ~280px, все равной ширины. */
}
```

### Навбар: логотип, центрированные ссылки, действия справа

```css
.navbar {
  display: flex;
  align-items: center;
  padding: 0 24px;
}

.logo {
  flex: none;
  margin-right: auto; /* выталкивает всё остальное вправо */
}

.nav-links {
  display: flex;
  gap: 24px;
  /* Центрируются между логотипом и действиями:
     логотип имеет margin-right: auto, действия — margin-left: auto */
  margin: 0 auto;
}

.nav-actions {
  flex: none;
  margin-left: auto; /* выталкивает себя в крайний правый край */
}
```

### Колонки одинаковой высоты с разным контентом

```css
.columns {
  display: flex;
  align-items: stretch; /* по умолчанию — колонки растягиваются до высоты самой высокой */
  gap: 24px;
}

.column {
  flex: 1;
  min-width: 0;
}
```

`align-items: stretch` (значение по умолчанию) растягивает все flex-элементы до размера самого высокого по поперечной оси. Именно поэтому Flexbox автоматически создаёт колонки одинаковой высоты — раскладка, для которой раньше требовались хаки с `display: table-cell`.

## Типичные ошибки на интервью

**"Почему `flex-grow: 1` у всех элементов не делает их одинаковой ширины?"**

Делает — но только если `flex-basis` одинаков у всех элементов. С `flex-basis: auto` (значение по умолчанию в `flex: 0 1 auto`) элементы начинают с размера своего контента и лишь *оставшееся* свободное пространство делят поровну. Элемент с бо́льшим контентом начинает больше и остаётся больше. Для настоящей равной ширины: `flex: 1` (устанавливает `flex-basis: 0%`) — все начинают с нуля и растут одинаково.

---

**"Элемент с `flex-shrink: 1` переполняет контейнер — почему?"**

`min-width: auto` у flex-элементов. Браузер вычислил минимальный размер контента элемента (например, длинное неразрывное слово или изображение) и не позволяет сжать его ниже — независимо от `flex-shrink`. Исправление: добавить `min-width: 0` к элементу, затем `overflow: hidden` (или `auto`) для управления содержимым.

---

**"В чём разница между `flex: 1` и `flex: 1 1 auto`?"**

Значение `flex-basis`. `flex: 1` разрешается в `flex: 1 1 0%` — элементы начинают с нуля и растут пропорционально, давая равные размеры. `flex: 1 1 auto` использует ширину контента каждого элемента как стартовую точку, поэтому элементы с бо́льшим контентом оказываются крупнее даже при одинаковых значениях `flex-grow`.

---

**"Как выровнять один flex-элемент вправо, оставив остальные слева?"**

`margin-left: auto` на элементе, который нужно справа. Во Flexbox `auto`-margin поглощает всё доступное свободное пространство в указанном направлении. `justify-self` не работает на flex-элементах (это свойство только для grid). Для выталкивания в конец: `margin-left: auto`. Для выталкивания в начало: `margin-right: auto`.

---

**"Почему `align-content` ничего не делает в моей flex-раскладке?"**

`align-content` влияет только на контейнеры с `flex-wrap: wrap`, у которых реально более одной строки элементов. В однострочном flex-контейнере (по умолчанию) `align-content` игнорируется — вместо него используйте `align-items`. Также: `align-content` требует, чтобы у контейнера была заданная высота, превышающая сумму flex-строк.

---

**"Flex-элемент содержит длинный текст, но `text-overflow: ellipsis` не работает."**

Flex-элемент недостаточно сжимается, чтобы запустить усечение. Нужны два исправления: (1) `min-width: 0` на flex-элементе (переопределяет значение по умолчанию `auto`, которое препятствует сжатию ниже размера контента), и (2) `white-space: nowrap; overflow: hidden; text-overflow: ellipsis` на текстовом элементе. Без `min-width: 0` flex-элемент вырастает до размера неусечённого текста, прежде чем усечение вообще рассматривается.
