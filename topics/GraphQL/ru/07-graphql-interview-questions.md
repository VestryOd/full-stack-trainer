# GraphQL Interview Questions

---

# 1. Что такое GraphQL?

GraphQL — язык запросов к API и runtime для их выполнения.

---

# 2. Почему появился GraphQL?

Чтобы решить проблемы:

- Overfetching
- Underfetching

REST API.

---

# 3. Чем GraphQL отличается от REST?

REST:

```txt
сервер определяет ответ
```

GraphQL:

```txt
клиент определяет ответ
```

---

# 4. Что такое Schema?

Контракт API.

Описывает типы,
поля и операции.

---

# 5. Что такое Resolver?

Функция,
которая получает данные для поля схемы.

---

# 6. Что получает Resolver?

```ts
(parent, args, context, info)
```

---

# 7. Что такое Query?

Операция чтения данных.

---

# 8. Что такое Mutation?

Операция изменения данных.

---

# 9. Что такое Subscription?

Realtime операция,
обычно работающая через WebSocket.

---

# 10. Что такое Input Type?

Тип для передачи сложных данных
в Mutation.

---

# 11. Что такое Fragment?

Переиспользуемый набор полей.

---

# 12. Что такое Variables?

Отдельная передача параметров запроса.

---

# 13. Что такое Introspection?

Возможность GraphQL
описывать собственную схему.

---

# 14. Почему GraphQL называют strongly typed?

Потому что схема строго описывает все типы данных.

---

# 15. Что такое N+1 Problem?

Один запрос на список сущностей
и отдельный запрос на каждую связанную сущность.

---

# 16. Как решить N+1?

DataLoader.

---

# 17. Что делает DataLoader?

- Batching
- Request-scoped caching

---

# 18. Почему DataLoader создают на каждый запрос?

Чтобы избежать:

- утечек памяти
- устаревших данных
- проблем безопасности

---

# 19. Что такое Query Complexity?

Оценка стоимости GraphQL запроса.

---

# 20. Что такое Depth Limiting?

Ограничение максимальной глубины запроса.

---

# 21. Зачем нужна Pagination?

Чтобы не загружать огромные объемы данных.

---

# 22. Почему Cursor Pagination лучше Offset?

Лучше масштабируется на больших таблицах.

---

# 23. Почему кеширование GraphQL сложнее?

Потому что все запросы идут через один endpoint.

---

# 24. Что такое Persisted Queries?

Предварительно зарегистрированные запросы,
идентифицируемые по hash.

---

# 25. Что такое Apollo Federation?

Механизм объединения нескольких GraphQL сервисов
в единую схему.

---

# 26. Что такое Gateway?

Сервис,
который объединяет несколько GraphQL API.

---

# 27. Что такое BFF?

Backend For Frontend.

Отдельный API слой,
оптимизированный под frontend.

---

# 28. Когда GraphQL лучше REST?

- сложный frontend
- mobile apps
- много связанных данных

---

# 29. Когда REST лучше GraphQL?

- простые CRUD системы
- публичные API
- высокая кешируемость

---

# 30. Самый популярный Senior вопрос

Как бы вы масштабировали большой GraphQL API?

Ответ:

Я бы использовал DataLoader для решения N+1, pagination для ограничения объема данных, query complexity и depth limiting для защиты от тяжелых запросов, persisted queries для кеширования и Apollo Federation для разделения схемы между микросервисами.
