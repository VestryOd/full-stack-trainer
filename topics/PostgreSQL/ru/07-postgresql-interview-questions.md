# PostgreSQL Interview Questions (Middle → Senior)

---

# 1. Что такое PostgreSQL?

### Ответ

PostgreSQL — это объектно-реляционная СУБД (RDBMS), которая поддерживает ACID-транзакции, MVCC, расширенные типы данных, индексы, JSONB и сложные SQL-запросы.

PostgreSQL подходит как для классических реляционных данных, так и для полуструктурированных данных через JSONB.

---

# 2. Что такое Primary Key?

### Ответ

Primary Key — это уникальный идентификатор строки.

Гарантирует:

- уникальность
- NOT NULL

Пример:

```sql
id UUID PRIMARY KEY
```

---

# 3. Что такое Foreign Key?

### Ответ

Foreign Key связывает таблицы между собой.

Пример:

```sql
user_id REFERENCES users(id)
```

Гарантирует ссылочную целостность.

То есть нельзя создать запись,
ссылающуюся на несуществующего пользователя.

---

# 4. Какие типы отношений бывают?

### Ответ

One-To-One

```txt
User → Profile
```

One-To-Many

```txt
User → Posts
```

Many-To-Many

```txt
Users ↔ Roles
```

Обычно через промежуточную таблицу.

---

# 5. Что такое нормализация?

### Ответ

Нормализация — это процесс устранения дублирования данных.

Например вместо:

```txt
post_author_name
post_author_email
```

используем:

```txt
posts.user_id
```

и отдельную таблицу users.

---

# 6. Что такое денормализация?

### Ответ

Осознанное дублирование данных ради производительности.

Например:

```txt
order.total_price
```

можно хранить в таблице,
вместо пересчета через JOIN.

---

# 7. Что такое транзакция?

### Ответ

Транзакция — это группа операций,
которая выполняется как единое целое.

Либо выполняются все операции:

```sql
COMMIT;
```

Либо ни одна:

```sql
ROLLBACK;
```

---

# 8. Что означает ACID?

### Ответ

Atomicity

Либо всё, либо ничего.

Consistency

Данные остаются корректными.

Isolation

Транзакции не мешают друг другу.

Durability

После COMMIT данные не потеряются.

---

# 9. Что такое MVCC?

### Ответ

MVCC (Multi-Version Concurrency Control) —
механизм конкурентного доступа PostgreSQL.

Вместо изменения строки создается новая версия строки.

Это позволяет:

```txt
readers don't block writers
writers don't block readers
```

---

# 10. Почему PostgreSQL работает быстро под нагрузкой?

### Ответ

Основная причина — MVCC.

Чтение обычно не блокирует запись,
а запись не блокирует чтение.

Также помогают:

- индексы
- planner
- vacuum
- connection pooling

---

# 11. Что такое Dirty Read?

### Ответ

Чтение данных незавершенной транзакции.

PostgreSQL Dirty Reads не допускает.

Даже Read Uncommitted работает как Read Committed.

---

# 12. Что такое Non-Repeatable Read?

### Ответ

Один и тот же SELECT внутри транзакции
возвращает разные значения.

---

# 13. Что такое Phantom Read?

### Ответ

Повторный запрос возвращает новые строки,
которые появились после первого SELECT.

---

# 14. Какие уровни изоляции есть в PostgreSQL?

### Ответ

Read Committed

По умолчанию.

Repeatable Read

Все SELECT работают с одним snapshot.

Serializable

Максимальная изоляция.

---

# 15. Какой уровень изоляции используется по умолчанию?

### Ответ

Read Committed.

---

# 16. Чем Repeatable Read отличается от Read Committed?

### Ответ

Read Committed:

каждый SELECT может видеть новые коммиты.

Repeatable Read:

вся транзакция работает с одним snapshot данных.

---

# 17. Что такое Serializable?

### Ответ

Самый строгий уровень изоляции.

PostgreSQL делает вид,
будто транзакции выполнялись последовательно.

При конфликте одна транзакция завершается ошибкой:

```txt
could not serialize access
```

---

# 18. Что такое индекс?

### Ответ

Индекс — дополнительная структура данных,
которая ускоряет поиск строк.

Позволяет выполнять поиск:

```txt
O(log n)
```

вместо:

```txt
O(n)
```

---

# 19. Какой индекс PostgreSQL использует по умолчанию?

### Ответ

B-Tree.

---

# 20. Почему поиск по индексу быстрее?

### Ответ

Потому что B-Tree позволяет выполнять поиск логарифмически.

Для:

```txt
1 000 000 строк
```

понадобится примерно:

```txt
20 переходов
```

вместо полного обхода таблицы.

---

# 21. Какие индексы PostgreSQL вы знаете?

### Ответ

B-Tree

Основной индекс.

GIN

JSONB, Full Text Search.

GiST

Геоданные.

Hash

Поиск по равенству.

BRIN

Очень большие таблицы.

---

# 22. Что такое Composite Index?

### Ответ

Индекс по нескольким колонкам.

Пример:

```sql
(name, age)
```

---

# 23. Что такое Left Prefix Rule?

### Ответ

Для индекса:

```sql
(name, age)
```

индекс работает для:

```sql
WHERE name = ...
```

и

```sql
WHERE name = ... AND age = ...
```

Но обычно неэффективен для:

```sql
WHERE age = ...
```

---

# 24. Почему PostgreSQL может не использовать индекс?

### Ответ

Planner считает стоимость.

Если выборка слишком большая,
индекс может оказаться медленнее,
чем полный проход таблицы.

---

# 25. Что такое EXPLAIN?

### Ответ

Показывает план выполнения запроса.

---

# 26. Чем EXPLAIN отличается от EXPLAIN ANALYZE?

### Ответ

EXPLAIN показывает предполагаемый план.

EXPLAIN ANALYZE реально выполняет запрос
и показывает фактические значения.

---

# 27. Что такое Seq Scan?

### Ответ

Sequential Scan.

Полный проход по таблице.

---

# 28. Что такое Index Scan?

### Ответ

Поиск через индекс.

Обычно используется для точечных запросов.

---

# 29. Что такое Index Only Scan?

### Ответ

PostgreSQL получает данные только из индекса,
не читая саму таблицу.

Это один из самых быстрых вариантов выполнения.

---

# 30. Что такое Bitmap Scan?

### Ответ

Гибридный вариант.

Используется когда совпадает большое количество строк.

---

# 31. Что такое VACUUM?

### Ответ

Очищает dead tuples,
оставшиеся после работы MVCC.

---

# 32. Что такое Autovacuum?

### Ответ

Фоновый процесс PostgreSQL,
который автоматически запускает VACUUM и ANALYZE.

---

# 33. Почему нельзя отключать Autovacuum?

### Ответ

Появляются:

- dead tuples
- table bloat
- index bloat

Со временем производительность сильно падает.

---

# 34. Что такое dead tuple?

### Ответ

Старая версия строки,
которая больше не нужна транзакциям.

---

# 35. Что такое table bloat?

### Ответ

Разрастание таблицы за счет накопления dead tuples.

---

# 36. Что такое FOR UPDATE?

### Ответ

Берет row-level lock.

Пример:

```sql
SELECT *
FROM accounts
WHERE id = 1
FOR UPDATE;
```

Используется для предотвращения race conditions.

---

# 37. Что такое deadlock?

### Ответ

Две транзакции ждут друг друга.

Пример:

```txt
A держит row1
ждет row2

B держит row2
ждет row1
```

PostgreSQL автоматически завершает одну из них.

---

# 38. Что такое JSONB?

### Ответ

Бинарный JSON формат PostgreSQL.

Поддерживает:

- индексацию
- быстрый поиск
- операции над JSON

Обычно предпочтительнее JSON.

---

# 39. Что такое Connection Pool?

### Ответ

Механизм переиспользования соединений с БД.

Создание подключения дорого,
поэтому приложения используют пул соединений.

---

# 40. Самый частый Senior вопрос

Почему PostgreSQL способен обрабатывать большое количество параллельных транзакций без тотальных блокировок?

### Ответ

Благодаря MVCC.

UPDATE создает новую версию строки,
а не изменяет существующую.

Поэтому чтение и запись обычно не блокируют друг друга,
а старые версии строк очищаются через VACUUM.
