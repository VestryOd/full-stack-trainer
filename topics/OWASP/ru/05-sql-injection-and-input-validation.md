# SQL Injection и Input Validation

## Одна из старейших атак

Но до сих пор встречается.

---

# Что такое SQL Injection

Атака,
при которой пользователь
влияет на SQL запрос.

---

# Плохой пример

```ts
const query =

`SELECT *
 FROM users
 WHERE email='${email}'`;
```

---

Пользователь вводит:

```txt
' OR 1=1 --
```

---

Получаем:

```sql
SELECT *
FROM users
WHERE email=''
OR 1=1
```

---

Результат:

```txt
все пользователи
```

---

# Более опасный пример

```txt
DROP TABLE users
```

---

# Почему это работает

Потому что пользовательский ввод
смешивается с SQL.

---

# Parameterized Query

Главная защита.

---

Правильно:

```ts
db.query(
 "SELECT * FROM users WHERE email=$1",
 [email]
);
```

---

Теперь:

```txt
SQL

и

данные
```

---

Разделены.

---

---

Любой может отправить:

```http
curl

Postman

Swagger
```

---

# Валидация нужна на Backend

Всегда.

---

---


---

# Mass Assignment

Очень любят спрашивать.

---

Проблема.

---

Есть модель:

```ts
User
```

---

Пользователь отправляет:

```json
{
 "name":"Max",
 "role":"admin"
}
```

---

Если слепо сохранить:

```ts
db.create(req.body)
```

---

Получаем:

```txt
эскалацию привилегий
```

---

# Решение

Использовать:

```txt
DTO

Whitelist

Field Mapping
```

---

# Частый вопрос

Защищает ли ORM от SQL Injection?

Ответ:

В большинстве случаев да.

---

Но raw queries всё еще могут быть уязвимы.

---

# Частый вопрос

Почему нельзя доверять Frontend Validation?

Ответ:

Потому что клиент можно обойти и отправить любой запрос напрямую.

---

# Частый вопрос

Что такое Mass Assignment?

Ответ:

Уязвимость, при которой пользователь может передать поля, которые не должен иметь право изменять.

---

# Частый вопрос

Что важнее: Validation или Sanitization?

Ответ:

Обе.

Validation проверяет корректность данных, а Sanitization делает данные безопасными.

---

# Interview Answer

SQL Injection возникает при смешивании пользовательского ввода и SQL-кода. Основная защита — параметризованные запросы и ORM. Кроме этого, сервер всегда должен выполнять валидацию и санитизацию входящих данных независимо от проверок на фронтенде.
